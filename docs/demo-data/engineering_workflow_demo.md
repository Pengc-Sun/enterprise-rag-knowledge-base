# Engineering Workflow Demo

## Pull Request Review

Production code changes require at least one peer review and passing CI checks before merge. Security-sensitive changes require an additional approval from the owning team.

## Incident Fixes

Emergency fixes may be merged with expedited review when an incident commander approves the change. A follow-up review and test coverage update are required within two business days.

## Release Notes

Each release should include user-visible changes, migration notes, known limitations, and rollback instructions.
